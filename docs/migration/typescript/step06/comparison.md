# Step 06 Comparison - Settings Webview Split

## Scope

- Added typed settings webview modules under `src/webview/settings`.
- Kept `src/settingsWebview.js` as the compatibility inline HTML/CSS/JS renderer.
- Moved state serialization and CSP string construction behind typed helpers.
- Added stable contracts for settings page ids, VS Code webview style token names, and client `postMessage` names.
- Updated DOM tests to compile first and exercise `out/src/settingsWebview.js`, matching the runtime package entry model.
- Updated `.codex` maps and the docs drift guard so `src/webview` is a tracked source anchor.

## Behavior Notes

- No webview selector, page id, CSP nonce, message type, or UI label was intentionally changed.
- The split is intentionally conservative. Full extraction of the inline CSS, markup, and browser script remains a later migration step.
- The compatibility renderer still owns the rendered document, while TypeScript modules now own the reusable contracts around the renderer.
- `.codex/change-impact.md` was already modified outside this step and remains intentionally unstaged.
